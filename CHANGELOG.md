# 更新记录 · 人生海海 PWA

每次改动都记录在这里：做了什么、为什么、影响到哪。最新的在最上面。
（更早的历史见 git 提交记录。）

## 1.5.4 — 2026-06-20

**为家人使用做适配：去个人化 + 扩词库 + 老人健康预警**

- 去掉写死的「小瑞瑞/瑞瑞 → 儿子」，家人用不再被误判为「儿子」。
- 人物识别扩展：新增 奶奶/爷爷/外婆/外公、孙辈、兄弟姐妹、配偶、医护、邻居/老乡。
- 主题词库大幅扩充，并新增「财务」「情绪心情」两类；调整优先级，避免「吃」「累」等通用词抢标签（如「和客户吃饭谈项目」现归「工作」而非「饮食」）。
  - 健康新增：血压/血糖/吃药/复查/体检/理疗/挂号/住院/手术…（覆盖老人常写的内容）
- 事务/自身记录词库扩充：就医办事（挂号/复查/缴费/养老金…）、自身健康（血压/膝盖/记性/孤独…）。
- **新增老人高危预警**：跌倒（摔倒/滑倒/绊倒）、疑似中风（嘴歪/说话不清/半边无力/手脚发麻）。仅作即时提醒，不替代医疗判断；「没有/未出现」等否定表述不会误报。
- 仅对**新记录**生效，不改动已有记录的归类（未升 ARCHIVE_VERSION，避免覆盖你手动改过的分类）。

## 1.5.3 — 2026-06-20

**搜索体验优化**

- 搜索框不再每敲一个字就整页重渲染并强制重新聚焦。改为只更新结果区，输入框的焦点、光标位置都保留。
- 修复中文输入法问题：以前组字过程中触发整页重渲染，会打断拼音、吞字；现在组字时不刷新，确认上屏后才更新结果。
- 顺带消除了每次按键都重新绑定全部事件监听的浪费。
- 原因：原实现 `render()` + `focus()` 放在每次 `input` 上，对中文输入和性能都不友好。

## 1.5.2 — 2026-06-20

**备份与恢复改版**

- 新增「一键备份到 iCloud」：导出普通 `.json`，**免密码**，依托 iCloud 自身加密，设为默认推荐方式。
- 修复 iOS 上「选不中备份文件、无法恢复」：去掉文件选择器的 `.haihai` 后缀过滤，`.json` 与 `.haihai` 都能选中。
- 恢复时自动识别文件类型：免密码备份直接恢复，加密备份才要求输入密码。
- 保留「加密备份（设密码）」作为额外保护选项；旧的 `.haihai` 加密备份仍可凭原密码恢复，向后兼容。
- 原因：用户反馈每次备份都要设/记密码太繁琐，且忘记密码会导致备份永久打不开；自用日记存自己 iCloud 时，额外密码多为冗余。
