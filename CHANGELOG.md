## 1.5.6 — 2026-06-20

**界面「Codex 分析」改为「Claude」+ 电脑桥默认用 Claude Opus 4.8**

- 总结页/连接页所有面向用户的文案由 Codex 改为 Claude（如「CLAUDE · 整体分析」「用 Claude 更新整体分析」「连接电脑上的 Claude」等）。代码内部标识符（`codexConfig`、`data-codex-analyze`、`source==="codex"` 兜底判断等）保持不变，不影响逻辑。
- 电脑桥启动脚本默认 `HAIHAI_ANALYZER=claude`、`HAIHAI_CLAUDE_MODEL=claude-opus-4-8`；可用环境变量改回 codex 或换模型。
- 模型说明：当前最新 Opus 即 4.8（`claude-opus-4-8`，已实测可用）；"4.6"是 Sonnet 的版本号，不是 Opus。

## 电脑桥 bridge — 2026-06-20（非 PWA 版本，仅本机生效）

**整体分析引擎支持切换 Codex / Claude Code CLI**

- `bridge/run-analysis.mjs` 新增 Claude 后端：设 `HAIHAI_ANALYZER=claude` 即用本机 `claude -p` 跑整体分析（默认仍是 `codex`，不影响现状）。可选 `HAIHAI_CLAUDE_MODEL` 指定模型。
- Claude 无 `--output-schema` 约束解码，故把 JSON Schema 写进提示词强约束，并加「解析失败再修复一次」的兜底；实测中文 JSON 直接通过。
- 修了一个会毁掉中文输出的隐患：子进程 stdout 用 `setEncoding("utf8")`，避免多字节字符跨 chunk 被切碎成乱码。
- 安全沿用：空目录运行、禁用全部工具与 MCP、只读；记忆视为不可信输入。
- 分析结果新增 `engine` 字段（codex/claude）；`source` 仍保持 `codex` 以兼容 App 端「来自电脑桥」的判断。

## 1.5.5 — 2026-06-20

**自动高频主题（替代"关注点"问卷）+ 电脑桥提示词去个人化**

- 分类页、总结页顶部自动显示「最近你记得最多的是 **X** · N 条」。不靠开屏问卷，纯靠已有数据观察得出——千人千面、零门槛，老人也无需设置。
  - 取出现最多的有信息量主题，排除兜底的"日常"，且需累计 ≥3 条才显示，避免数据太少时乱报。
  - 新增 `.focus-line` 样式，适配浅色/深色两种页面。
- 电脑桥（bridge）提示词同步去个人化：删掉"小瑞瑞/瑞瑞→儿子"的写死规则，改为"以记录原文出现的称呼为准"，与 App 端 1.5.4 保持一致，家人用不再被误判。

## 1.5.4 — 2026-06-20

**为家人使用做适配：去个人化 + 扩词库 + 老人健康预警**

- 去掉写死的「小瑞瑞/瑞瑞 → 儿子」，家人用不再被误判为「儿子」。
- 人物识别扩展：新增 奶奶/爷爷/外婆/外公、孙辈、兄弟姐妹、配偶、医护、邻居/老乡。
- 主题词库大幅扩充，并新增「财务」「情绪心情」两类；调整优先级，避免「吃」「累」等通用词抢标签（如「和客户吃饭谈项目」现归「工作」而非「饮食」）。
  - 健康新增：血压/血糖/吃药/复查/体检/理疗/挂号/住院/手术…（覆盖老人常写的内容）
- 事务/自身记录词库扩充：就医办事（挂号/复查/缴费/养老金…）、自身健康（血压/膝盖/记性/孤独…）。
- **新增老人高危预警**：跌倒（摔倒/滑倒/绊倒）、疑似中风（嘴歪/说话不清/半边无力/手脚发麻）。仅作即时提醒，不替代医疗判断；「没有/未出现」等否定表述不会误报。
- 仅对**新记录**生效，不改动已有记录的归类（未升 ARCHIVE_VERSION，避免覆盖你手动改过的分类）。

## 1.5.3 — 2026-06-20

**搜索体验优化**

- 搜索框不再每敲一个字就整页重渲染并强制重新聚焦。改为只更新结果区，输入框的焦点、光标位置都保留。
- 修复中文输入法问题：以前组字过程中触发整页重渲染，会打断拼音、吞字；现在组字时不刷新，确认上屏后才更新结果。
- 顺带消除了每次按键都重新绑定全部事件监听的浪费。
- 原因：原实现 `render()` + `focus()` 放在每次 `input` 上，对中文输入和性能都不友好。

## 1.5.2 — 2026-06-20

**备份与恢复改版**

- 新增「一键备份到 iCloud」：导出普通 `.json`，**免密码**，依托 iCloud 自身加密，设为默认推荐方式。
- 修复 iOS 上「选不中备份文件、无法恢复」：去掉文件选择器的 `.haihai` 后缀过滤，`.json` 与 `.haihai` 都能选中。
- 恢复时自动识别文件类型：免密码备份直接恢复，加密备份才要求输入密码。
- 保留「加密备份（设密码）」作为额外保护选项；旧的 `.haihai` 加密备份仍可凭原密码恢复，向后兼容。
- 原因：用户反馈每次备份都要设/记密码太繁琐，且忘记密码会导致备份永久打不开；自用日记存自己 iCloud 时，额外密码多为冗余。
