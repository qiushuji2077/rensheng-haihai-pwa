# 人生海海 PWA

这是原生 SwiftUI 项目的手机网页版本，适合直接安装到 iPhone 主屏幕长期自用。

## 本地预览

在 `web` 文件夹启动任意静态服务器，例如：

```bash
python3 -m http.server 4173
```

访问 `http://localhost:4173`。

## iPhone 安装

正式安装需要把这个文件夹部署到支持 HTTPS 的静态网站。使用 Safari 打开网址后：

1. 轻点“分享”。
2. 选择“添加到主屏幕”。
3. 轻点“添加”。

记忆保存在当前设备本地，不需要账户。

## 数据耐久（重要）

为避免 iOS 长期不用清理网页存储而丢记忆，本版做了三重保护：

- **双写**：每条记忆同时写入 `localStorage` 与 **IndexedDB**（容量更大、更耐久），两者互为镜像。
- **启动兜底恢复**：每次打开会从 IndexedDB 把记忆并集补回 `localStorage`；即便一边被系统清理，也能自动找回。
- **持久化授权**：启动时调用 `navigator.storage.persist()`，尽量让本站存储免于被自动清除。
- 写入失败不再静默——会明确提示「请尽快创建一次备份」。

即便如此，**最稳的长期保险仍是定期用「记忆保护」导出加密备份**（存到「文件」/iCloud Drive），
或连接电脑桥让记忆同步到电脑本地镜像。

## 分类与整体分析

- 单条记录只做轻量归档：人、事、内容，以及人物和主题。
- “小瑞瑞 / 瑞瑞”统一归到“人 · 儿子”。
- 不再逐条生成空泛的观点提示；关键线索、总结和方向只结合全部记录生成。
- 明确的高危词会在手机端即时提醒，但不会替代医疗或安全诊断。
- 连接电脑桥后，记录会同步到电脑中的加密镜像，再由已登录的 Codex 做整体分析。

## 备份与恢复

首页右上角的盾牌进入“记忆保护”，提供两种备份方式：

- **一键备份到 iCloud（免密码，推荐日常使用）**：导出普通 `.json` 文件，通过系统分享存到「文件」/iCloud Drive，安全性依托 iCloud 自身的加密。无需设置或记忆密码。
- **加密备份（设密码，额外保护）**：导出 `.haihai` 文件，适合需要把备份存到不那么可信的位置时使用。
  - 使用 PBKDF2-SHA-256（310,000 次）派生密钥，AES-256-GCM 加密并校验完整性。
  - 密码只在当前操作中使用，不保存、不上传，也无法找回。

恢复时：

- 文件选择器不再按后缀过滤，`.json` 与 `.haihai` 都能选中（修复 iOS 上自定义后缀文件变灰、选不中的问题）。
- 自动识别文件类型：免密码备份直接恢复；加密备份才会要求输入密码。
- 默认恢复方式为“合并”，避免覆盖当前手机上的新记忆。
- 备份包含整体分析结果；iCloud 文件仍只用于灾难恢复，不交给定时任务扫描。

## Codex 电脑桥

1. 双击项目根目录的 `启动人生海海桥.command`。
2. 在打开的配对页复制连接码。
3. 在 App 的“总结与方向”里选择连接 Codex，粘贴一次连接码。

桥接地址会自动发现。电脑桥只开放记忆同步与固定的整体分析，不接收自由指令。
电脑中的同步镜像使用 AES-256-GCM 再次加密保存。

Codex App 中的“人生海海每周整体分析”默认每周日 20:00 运行。电脑需开机且
Codex 正在运行；若超过 14 天没有从手机同步，任务不会用陈旧数据覆盖旧总结。

## 更新策略

- 联网时优先读取线上最新版，断网时才使用本地缓存。
- Service Worker 更新不使用浏览器 HTTP 缓存，并在控制器切换后自动刷新。
- “记忆保护”底部可查看版本号并手动检查更新。
